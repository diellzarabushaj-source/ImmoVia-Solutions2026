Please audit the entire ImmoVia365 platform and make sure all dashboards, roles, routes, data, forms, buttons, statuses and actions are fully synchronized and working together.

Do not redesign the platform from zero. Keep the existing structure, design and logic as much as possible. Your task is to connect everything properly, fix broken or incomplete functionality, remove inconsistencies, and make the full platform work smoothly end-to-end.

The platform has three main role experiences:
1. Admin
2. Project Poster
3. Service Provider

Make sure each role dashboard is connected to the same real data and that actions in one area correctly update the other related areas.

For example:
- When a Project Poster creates or publishes a project, it must appear correctly in their dashboard, in the Admin dashboard, and for matching Service Providers.
- When a Service Provider sends an offer, it must appear under that provider’s offers, under the related project for the Project Poster, and inside the Admin offer/project management area.
- When a Project Poster accepts, rejects or shortlists an offer, the status must update everywhere.
- When Admin verifies, rejects, suspends or edits a Service Provider, that status must update in the provider dashboard and wherever provider information is shown.
- When Admin edits, deletes, publishes or changes a project/category/blog/user, the update must be reflected everywhere it is used.
- Notifications, counters, dashboard statistics, badges and activity lists must always match the actual database state.
