MARKETPLACE ACCOUNT STRUCTURE:

The marketplace has 2 main account types and 4 subtypes.

Main account type 1:
project_poster

This is a user who posts projects, requests services, or looks for workers/providers.

Subtypes:
- individual
- company

Main account type 2:
service_provider

This is a user who looks for work/projects, applies to projects, or offers services.

Subtypes:
- individual
- company

Correct examples:
1. account_type = project_poster, account_subtype = individual
2. account_type = project_poster, account_subtype = company
3. account_type = service_provider, account_subtype = individual
4. account_type = service_provider, account_subtype = company

Admin is separate.

Use this structure:
role = admin or user
account_type = project_poster or service_provider
account_subtype = individual or company
status = draft, pending_review, approved, rejected, suspended

Do not mix role with account_type.
Do not make admin an account_type.
Do not create account types like individual_project_poster or company_service_provider.
Use account_type + account_subtype instead.

APPROVAL LOGIC:

Use status clearly.

For profiles:
draft = user is still editing
pending_review = waiting for admin approval
approved = visible publicly
rejected = hidden
suspended = hidden and blocked from public activity

For projects:
draft = user is still editing
pending_review = waiting for admin approval
open = approved and public
in_progress = provider selected/work started
completed = finished
cancelled = cancelled
rejected = hidden
