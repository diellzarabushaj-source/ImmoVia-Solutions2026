Create a complete MVP Service Provider Dashboard for ImmoVia365.

ImmoVia365 is a Swiss marketplace that connects project owners with handymen, renovation companies, construction companies and service providers in Switzerland.

The dashboard must be designed only for users with the role “Service Provider / Anbieter”.

Customers / Project Posters must not see this dashboard. Customers have their own separate dashboard for creating projects, viewing applications, messages and offers.

The Service Provider Dashboard should be professional, clean, Swiss-style, easy to understand and suitable for an MVP.

Use German as the default language for all visible text.

The platform should also support language switching:

* Deutsch
* English
* Shqip
* Français

Design style:

* Clean Swiss marketplace look
* White or very light background
* Dark navy text
* Bright blue accent color
* Rounded cards
* Simple icons
* Clear dashboard layout
* Mobile responsive
* Professional and trustworthy

MAIN PURPOSE OF SERVICE PROVIDER DASHBOARD

The Service Provider Dashboard should allow companies and individual providers to:

* Create and manage their provider profile
* Add services and categories
* Add service areas / cities
* Upload portfolio images
* Find matching projects
* Apply to projects
* Send and receive messages
* Send offers inside conversations
* Manage applications
* See current subscription plan
* See remaining monthly applications
* See visibility / ranking status
* See provider badge
* Upgrade plan
* Manage billing
* Manage reviews

USER ROLE RULE

If user.role === “service_provider”, show this dashboard.

If user.role === “customer”, do not show:

* Provider dashboard
* Provider pricing
* Provider upgrade options
* Application limits
* Provider visibility settings
* Provider billing
* Provider badges management

FINAL SERVICE PROVIDER DASHBOARD FLOW

1. Service Provider registers.
2. Service Provider creates profile.
3. Service Provider chooses categories and city/service area.
4. Service Provider uploads portfolio images.
5. Service Provider sees current plan and application limit.
6. Service Provider browses matching projects.
7. Service Provider sends application or message to customer.
8. System creates project-based conversation.
9. Provider and customer communicate inside ImmoVia365.
10. Provider can send an offer inside conversation.
11. Customer can accept or decline offer.
12. Provider can track applications, messages, offers and plan status.
13. Paid providers receive higher visibility and better badges.
14. Free providers remain visible but appear lower in relevant listings.

The dashboard should be simple, clear, professional and ready for MVP launch.
