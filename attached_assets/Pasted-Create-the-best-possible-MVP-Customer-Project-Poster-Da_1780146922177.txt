Create the best possible MVP Customer / Project Poster Dashboard for ImmoVia365.

ImmoVia365 is a Swiss marketplace that connects people who need renovation, construction, home improvement or property services with professional Service Providers in Switzerland.

This dashboard is ONLY for users with the role “Customer” or “Project Poster”.

Service Providers must have a completely separate dashboard. Customers must never see provider pricing plans, provider subscription options, provider billing, provider upgrade buttons, provider ranking controls or provider application limits.

The dashboard must be simple, professional, trustworthy and very easy to use.

Default language:
German

Supported languages:
Deutsch
English
Shqip
Français

All visible UI text must be translatable. German is the default language.

Design style:
Clean Swiss marketplace design.
White or very light background.
Dark navy typography.
Bright blue accent color.
Rounded cards.
Clear buttons.
Simple icons.
Mobile responsive.
Professional but not complicated.

MAIN GOAL

The Customer Dashboard should help a customer do these things easily:

1. Create a project for free.
2. Upload photos and files.
3. Receive applications from Service Providers.
4. Compare providers.
5. Send messages to Service Providers.
6. Receive messages from Service Providers.
7. Reply to provider messages.
8. Receive photos, PDFs and documents from providers.
9. Send photos, PDFs and documents to providers.
10. Receive offers.
11. Accept or decline offers.
12. Save favourite providers.
13. Close or archive projects.
14. Leave a review after project completion.

The main customer flow should be:

Post project → receive applications → receive messages → reply to providers → compare offers → choose provider.

Customer clicks “Nachricht senden” to reply to a provider application.

ACCESS AND SECURITY RULES

- Customer can only view their own projects.
- Customer can only view conversations where they are a participant.
- Customer can only view offers connected to their own projects.
- Users cannot access projects, conversations or offers by guessing IDs.
- Files must be validated by type and size.
- Do not allow executable files.
- Admin can view reported conversations only for support and moderation.
- The messaging system should not expose private phone numbers or emails automatically.
- Communication should stay inside ImmoVia365 unless the user chooses otherwise.
