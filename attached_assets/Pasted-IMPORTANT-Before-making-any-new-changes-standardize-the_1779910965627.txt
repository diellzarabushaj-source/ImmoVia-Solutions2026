IMPORTANT: Before making any new changes, standardize the entire marketplace terminology and structure.

The platform must use ONLY these exact names everywhere in the app, database, admin dashboard, main page, onboarding, filters, buttons, public profiles, and code logic.

Do not use different names like client, customer, freelancer, worker, job seeker, employer, provider, contractor, talent, company user, business user, etc.

Use only this simple structure:

MAIN ROLE 1:
PROJECT POSTER

Meaning:
A Project Poster is a person or company that posts a project because they need workers, service providers, companies, or professionals.

PROJECT POSTER has 2 subtypes:
1. Individual Project Poster
2. Company Project Poster

MAIN ROLE 2:
SERVICE PROVIDER

Meaning:
A Service Provider is a person or company that offers work, applies to projects, or wants to get hired.

SERVICE PROVIDER has 2 subtypes:
1. Individual Service Provider
2. Company Service Provider

So the marketplace has:

1. Project Poster
   - Individual Project Poster
   - Company Project Poster

2. Service Provider
   - Individual Service Provider
   - Company Service Provider

Use these exact database values:

role:
- admin
- user

account_type:
- project_poster
- service_provider

account_subtype:
- individual
- company

Do not create any other account_type values.
Do not create 4 main roles.
Do not mix admin with account_type.
Admin is only role = admin.

Correct examples:

Example 1:
role = user
account_type = project_poster
account_subtype = individual
Display name: Individual Project Poster

Example 2:
role = user
account_type = project_poster
account_subtype = company
Display name: Company Project Poster

role = user

Example 4:

Display name: Company Service Provider

Example 5:
role = admin
Admin can manage the platform, but admin is not a marketplace account_type.

PROJECT LOGIC:

Only Project Posters can create projects.

Rule:
if account_type = project_poster, user can create projects.
if account_type = service_provider, user cannot create projects but can apply to projects.

Project statuses:
- draft
- pending_review
- open
- in_progress
- completed
- cancelled
- rejected

When a Project Poster creates a project, it becomes pending_review.
Admin must approve the project before it becomes public.
Approved projects become open.
Only open projects are visible publicly.

APPLICATION LOGIC:

Only Service Providers can apply to projects.

Rule:
if account_type = service_provider, user can apply to open projects.
if account_type = project_poster, user cannot apply to projects.

PROFILE LOGIC:

All users can have a profile.

Profile statuses:
- draft
- pending_review
- approved
- rejected
- suspended

Only approved profiles are public.

Individual profile fields:
- full name
- profession
- skills
- experience
- location
- phone
- email

- profile image

Company profile fields:
- company name
- company logo
- services
- category
- location
- phone
- email
- website
- description
- portfolio/images

PUBLIC VISIBILITY RULES:

Only show approved profiles publicly.
Only show open projects publicly.

Hide:
- draft
- pending_review
- rejected
- suspended
