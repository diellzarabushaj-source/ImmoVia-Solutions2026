Create a complete Service Provider dashboard monetization system for ImmoVia365.

ImmoVia365 is a Swiss marketplace that connects project owners with handymen, renovation companies, construction companies and service providers.

Important:
This monetization system must be visible only to users with the role “Service Provider”.
Do not show pricing plans, upgrade options, contact visibility controls or application limits to normal customers / project posters.
Customers should only see project creation, their own projects, applications and provider profiles.
Only Service Providers should see subscription plans, upgrade buttons, application limits, contact visibility status, billing and provider visibility features.

User roles:

1. Customer / Project Poster
   This user can:

* Create projects
* Edit projects
* Upload project photos
* View provider applications
* Compare provider profiles
* Send inquiries
* Close projects

This user must NOT see:

* Provider pricing plans
* Upgrade plan buttons
* Contact visibility settings
* Application limits
* Provider badges management
* Billing for provider plans
* Featured provider options

2. Service Provider / Anbieter
   This user can:

* Create and manage provider profile
* Add company or individual provider information
* Add services and categories
* Upload portfolio images
* View available projects
* Apply to projects
* Manage applications
* See monthly application limit
* See current subscription plan
* Upgrade subscription plan
* Manage contact visibility
* Manage billing
* See provider badge
* See visibility status

Dashboard behavior:
When a logged-in user has role “Service Provider”, show a dedicated Provider Dashboard with monetization sections.
When a logged-in user has role “Customer”, hide all monetization sections completely.

Service Provider Dashboard navigation should include:

1. Übersicht
2. Mein Anbieterprofil
3. Projekte finden
4. Meine Bewerbungen
5. Mein Plan
6. Sichtbarkeit
7. Bewertungen
8. Rechnungen
9. Einstellungen

The “Mein Plan” section should be visible only for Service Providers.

* Email should be stored in the system but not shown publicly.

Provider project application behavior:

When a Service Provider applies to a project:

* Check provider plan
* Check remaining monthly applications
* If applications are available, allow application
* Reduce remaining monthly application count by 1
* Save application in “Meine Bewerbungen”
* Notify the customer / project poster

If the provider has reached the monthly limit:

* Do not allow application
* Show upgrade prompt

Upgrade prompt:
“Mehr Projekte erreichen? Wechseln Sie auf einen höheren Plan und bewerben Sie sich auf mehr passende Projekte.”

Do not include in MVP:

* Pay-per-lead
* Lead credits
* Commission per completed project
* Escrow payments
* Complex bidding fees
* Advanced AI matching
* Complex ranking algorithm

MVP goal:
Keep the system simple, clean and easy to build.

The MVP monetization logic is:

* Customers post projects for free.
* Service Providers can register for free.
* Free Service Providers are limited.
* Paid Service Providers get visible contact details, more applications, more portfolio space and better visibility.
* Pricing and upgrade options are shown only inside the Service Provider Dashboard.
* Customer users never see provider pricing or subscription management.

Important UI rule:
If user.role === “service_provider”, show:

* Provider Dashboard
* Mein Plan
* Plan upgraden
* Bewerbungslimit
* Kontaktdaten sichtbar
* Rechnungen

If user.role === “customer”, hide all these sections completely.

Use German language for all visible frontend text.
Use CHF for all pricing.
Design should be clean, professional, Swiss, trustworthy and easy to understand.
