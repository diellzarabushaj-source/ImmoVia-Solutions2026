Aha, po ta kuptoj tash: **faqja po hapet në telefon**, por problemi është që **nuk po dalin projektet — po shfaq “0 Ergebnisse”**.

Pra nuk është problemi “mobile nuk hapet”, por:

**Mobile view loads, but project listings are empty. Desktop shows projects, mobile shows 0 results.**

Kjo zakonisht ndodh nga këto:

**1. Mobile po aplikon filter bosh gabimisht**
P.sh. `city = ""`, `category = ""`, `budget = ""`, por kodi e dërgon si filter dhe databaza kthen 0.

Shembull gabim:

```js
query.eq("city", city)
```

kur `city` është bosh.

Duhet:

```js
if (city && city.trim() !== "") {
  query.eq("city", city)
}
```

**2. Mobile ka default filter tjetër nga desktop**
P.sh. desktop shfaq “all projects”, ndërsa mobile ndoshta ka default `category`, `city`, `size`, ose `status` që nuk përputhet.

**3. API/data fetch ndryshe në mobile**
Mobile komponenti mund të përdorë tjetër function/API sesa desktopi.

P.sh. desktop përdor:

```js
fetchProjects()
```

mobile përdor:

```js
fetchFilteredProjects()
```

dhe kjo e dyta po kthen bosh.

**4. Supabase/RLS/Auth problem**
Në desktop ndoshta je logged in, në telefon jo. Nëse projektet janë të lejuara vetëm për logged-in user, telefoni si visitor i sheh 0.

**5. Projekti nuk është “public/active” për mobile**
Nëse mobile po kërkon vetëm:

```js
status = "published"
```

por në databazë statusi është:

```js
status = "active"
```

atëherë del 0.

- Registered service providers should see all projects.
