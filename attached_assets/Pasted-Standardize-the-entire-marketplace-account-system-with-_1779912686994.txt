Standardize the entire marketplace account system with ONE clear structure.

Do not create new role names.
Do not use client, freelancer, worker, employer, contractor, talent, customer, or job seeker.
Use only these official names everywhere:

1. Project Poster
2. Service Provider

Each has 2 subtypes:

Project Poster:
- Individual Project Poster
- Company Project Poster

Service Provider:
- Individual Service Provider
- Company Service Provider

DATABASE VALUES MUST BE:

role:
- admin
- user

account_type:
- project_poster
- service_provider

account_subtype:
- individual
- company

Admin is only role = admin.
Admin is NOT an account_type.
Admin must use and manage the same account_type and account_subtype structure as the public app.

Do not create 4 separate main roles.
Do not create values like individual_project_poster or company_service_provider.
The correct logic is always:
account_type + account_subtype.

Examples:
account_type = project_poster + account_subtype = individual = Individual Project Poster
account_type = project_poster + account_subtype = company = Company Project Poster
account_type = service_provider + account_subtype = individual = Individual Service Provider
account_type = service_provider + account_subtype = company = Company Service Provider

Admin actions must update the same fields used by the public app:

SYNC RULES:

If admin approves a profile:
profile status becomes approved
profile becomes public

If admin rejects a profile:
profile status becomes rejected
profile stays hidden

If admin suspends a profile:
profile status becomes suspended
profile stays hidden

If admin approves a project:
project status becomes open
project becomes public

If admin rejects a project:
project status becomes rejected
project stays hidden

If a Project Poster creates a new project:
project status becomes pending_review
project stays hidden until admin approval

If a user creates or updates a profile:
profile status becomes pending_review
profile stays hidden until admin approval
